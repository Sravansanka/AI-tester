How this was vibe-coded

The build started from a short, partly-truncated spec: five named techniques (hybrid embedding, hybrid vector search, reranking, query rewriting, generation) and a UI description — no pipeline step list, no data schema, no existing Jira-format template to copy. The rest was designed against that intent rather than filled in with guesses: a deterministic, combinatorial generator built the 5,000-row test-case corpus (component × sub-feature × scenario type × priority, not hand-authored, not random text), and the five pipeline stages were defined to match exactly what the spec's bullet list implied — rewrite feeds embed, embed feeds a fused hybrid retrieve, retrieve feeds rerank, rerank feeds a grounded, citation-required generate.

Everything was scaffolded, run, and verified stage by stage rather than all at once: the generator was validated for uniqueness before any embedding model was loaded; retrieval was smoke-tested directly against Qdrant before an HTTP layer existed; the reranker was checked for sane score reordering before generation was wired in. That order caught two real dependency conflicts early — a transformers version too new for the pinned reranker library, and a Qdrant client/server version mismatch — long before they could surface as a confusing failure deep inside a live chat request.
